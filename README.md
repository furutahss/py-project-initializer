# py-project-initializer

Pythonプロジェクト初期化ツールです。 

面倒なディレクトリ作成やGitの初期設定、仮想環境の構築をコマンド一発で自動化し、本質的なコーディングに即座に集中できる環境を提供します。

## 概要

このツールは、以下の作業を自動で実行します。

1. プロジェクトディレクトリの作成: 入力された名前で新しいフォルダを作成します。
2. Gitの初期化: git init を実行し、すぐにバージョン管理ができる状態にします。
3. 仮想環境 (venv) の構築: プロジェクトごとに独立したPython実行環境をセットアップします。
4. 雛形ファイルの生成: 以下のファイルを自動生成します。
  - `main.py`: プログラムのエントリーポイント。
  - `README.md`: 日付入りのプロジェクト概要。
  - `.gitignore`: `venv` やキャッシュファイルを除外する設定済みファイル。

## 使い方

1. 本リポジトリをクローンするか、`main.py` を任意の場所に保存します。
2. ターミナルで以下のコマンドを実行します。

```bash
python main.py [プロジェクト名]
```

※ プロジェクト名を省略して実行した場合は、対話形式で入力を求められます。

3. 作成されたディレクトリに移動し、開発を開始します。

```bash
cd [プロジェクト名]
# 仮想環境を有効化（macOS/Linuxの場合）
source venv/bin/activate
# main.pyを実行
python main.py
```

## プロジェクト構成（生成後）

実行後、以下のような構成でプロジェクトが作成されます。

```Plaintext
[プロジェクト名]/
├── venv/            # Python仮想環境
├── main.py          # メインスクリプト
├── README.md        # プロジェクトの説明（日付入り）
└── .gitignore       # Git管理除外設定
```

## 開発の動機（QOL向上）

「毎日手を動かす習慣」において最大の障害となるのは、開発を始める前の「環境構築という儀式」です。

このツールは、その儀式を5秒以内に終わらせることで、技術の積み重ね（草を生やすこと）をより楽しく、継続可能なものにすることを目的としています。
